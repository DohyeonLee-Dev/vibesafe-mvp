# VibeSafe

> Security scanner for vibe-coded projects — catch vulnerabilities in AI-generated code before they ship.

---

## What is VibeSafe?

VibeSafe is a command-line security scanner built specifically for **"vibe-coded" projects** — code generated by AI tools like Cursor, GitHub Copilot, Claude, ChatGPT, and others.

When you use AI to generate code, the output often contains subtle security issues that are easy to miss: hardcoded API keys, SQL injection vulnerabilities, disabled SSL verification, dangerous `eval()` calls, and more. You didn't write the code line by line, so you may not notice these problems during a quick review.

**VibeSafe solves this.** Point it at any project folder, and it scans every file for 20+ security patterns. In seconds, you get a clear report showing exactly what's wrong, where it is, and how to fix it.

### What it does

- Scans `.py`, `.js`, `.ts`, `.jsx`, `.tsx`, `.go`, `.html`, `.vue`, `.svelte`, `.yaml`, `.json`, `.env`, `.sh`, `.sql` and more
- Detects **hardcoded secrets**, **injection vulnerabilities**, **configuration mistakes**, **crypto weaknesses**, and **AI-specific anti-patterns**
- Outputs results as a **terminal table**, **JSON**, or a **visual HTML report**
- Includes a **web dashboard** (FastAPI) for browsing results in your browser
- Returns exit code 1 if critical/high issues are found — works in **CI/CD pipelines**
- Skips `node_modules`, `.git`, `__pycache__`, `venv`, `dist`, `build` and other non-essential directories automatically

### Who is it for?

- Solo developers shipping AI-generated code fast
- Indie hackers who don't have time for manual security audits
- Small teams using AI coding assistants in their workflow
- Anyone who wants a quick security check before pushing to production

## Usage

VibeSafe has three main commands: `scan`, `dashboard`, and `rules`.

### Command 1: `scan` — Scan a project for vulnerabilities

This is the main command. Point it at any folder containing source code.

#### Basic scan (terminal table output)

```bash
python -m vibesafe.cli scan /path/to/your/project
```

This prints a table to your terminal showing every security issue found, sorted by severity (critical first).

#### Generate an HTML report

```bash
python -m vibesafe.cli scan /path/to/your/project --format html --output report.html
```

Then open `report.html` in your browser. The report includes:
- A **letter grade** (A through F) based on the severity of findings
- **Summary counts** for critical, high, medium, and low issues
- Each finding with the **exact file path, line number, code snippet, and a fix suggestion**

#### Generate JSON output

```bash
python -m vibesafe.cli scan /path/to/your/project --format json --output results.json
```

Useful for piping into other tools or CI/CD systems. Each finding is a JSON object with `rule_id`, `severity`, `file_path`, `line_number`, `message`, `snippet`, and `fix_hint`.

#### Filter by severity

Only show high and critical issues:

```bash
python -m vibesafe.cli scan /path/to/your/project --severity high
```

Options: `low`, `medium`, `high`, `critical`. Showing `high` will include both `high` and `critical`.

#### Ignore specific rules

Skip rules you don't care about:

```bash
python -m vibesafe.cli scan /path/to/your/project --ignore VS-QLT-001 --ignore VS-QLT-002
```

You can repeat `--ignore` as many times as needed. Use `python -m vibesafe.cli rules` to see all rule IDs.

#### All options combined

```bash
python -m vibesafe.cli scan ./my-app \
  --format html \
  --output security-report.html \
  --severity medium \
  --ignore VS-QLT-001
```

### Command 2: `dashboard` — Launch a web dashboard

```bash
python -m vibesafe.cli dashboard /path/to/your/project --port 8000
```

This starts a local web server. Open **http://localhost:8000** in your browser to see an interactive security report.

The dashboard also exposes a JSON API:
- `GET /api/findings` — all findings as JSON
- `GET /api/summary` — summary counts (total, critical, high, medium, low)

Press `Ctrl+C` to stop the dashboard.

#### Change the port

```bash
python -m vibesafe.cli dashboard /path/to/your/project --port 3000
```

### Command 3: `rules` — List all security rules

```bash
python -m vibesafe.cli rules
```

Prints a table showing every rule ID, name, severity, and description. Use this to understand what VibeSafe checks for, and to find rule IDs for the `--ignore` flag.

## Security Rules Reference

VibeSafe includes 20+ built-in rules across 6 categories.

### Critical — Hardcoded Secrets

These are the most dangerous. Secrets in source code can be extracted by anyone with access to your repo.

| Rule ID | Name | What it catches |
|---------|------|----------------|
| VS-SEC-001 | Hardcoded API Key | `api_key = "sk-proj-abc123..."` |
| VS-SEC-002 | Hardcoded Password | `password = "super_secret"` |
| VS-SEC-003 | AWS Credentials | `AKIA...` access keys or `aws_secret_access_key` in code |
| VS-SEC-004 | Private Key | `-----BEGIN RSA PRIVATE KEY-----` embedded in files |

**How to fix:** Move all secrets to environment variables or a `.env` file (and add `.env` to `.gitignore`).

### High — Injection Vulnerabilities

Code that allows attackers to inject malicious input.

| Rule ID | Name | What it catches |
|---------|------|----------------|
| VS-INJ-001 | SQL Injection | `cursor.execute(f"SELECT * WHERE id = {input}")` |
| VS-INJ-002 | Command Injection | `os.system(f"rm {user_input}")` |
| VS-INJ-003 | XSS via innerHTML | `element.innerHTML = userInput` |
| VS-INJ-004 | eval() Usage | `eval(user_input)` in Python or JavaScript |

**How to fix:** Use parameterized queries for SQL, `subprocess` with list args for commands, `textContent` instead of `innerHTML`, and `JSON.parse()` instead of `eval()`.

### High — Dangerous AI-Generated Patterns

Patterns commonly introduced by AI code generators that create security holes.

| Rule ID | Name | What it catches |
|---------|------|----------------|
| VS-AI-001 | Pickle Deserialization | `pickle.loads(data)` — can execute arbitrary code |
| VS-AI-002 | YAML Unsafe Load | `yaml.load(data)` without `SafeLoader` |
| VS-AI-003 | Disabled SSL | `verify=False` or `rejectUnauthorized: false` |
| VS-CRY-002 | Hardcoded JWT Secret | `jwt.encode(payload, "hardcoded-secret")` |

**How to fix:** Use `json.loads()` instead of `pickle`, `yaml.safe_load()` instead of `yaml.load()`, always keep SSL verification enabled, and store JWT secrets in environment variables.

### Medium — Configuration Issues

Misconfigurations that weaken your application's security posture.

| Rule ID | Name | What it catches |
|---------|------|----------------|
| VS-CFG-001 | Debug Mode Enabled | `DEBUG = True` left in production code |
| VS-CFG-002 | CORS Allow All | `allow_origins=["*"]` allowing any domain |
| VS-CFG-003 | Missing HTTPS | `fetch("http://api.example.com/...")` |
| VS-CRY-001 | Weak Hash Algorithm | `hashlib.md5(...)` or `hashlib.sha1(...)` |

**How to fix:** Use environment variables for debug flags, restrict CORS to your domains, use HTTPS for all external URLs, and use SHA-256 or bcrypt.

### Low — Code Quality

Not security-critical, but worth cleaning up before shipping.

| Rule ID | Name | What it catches |
|---------|------|----------------|
| VS-QLT-001 | TODO/FIXME | `# TODO: fix this later` |
| VS-QLT-002 | Console.log | `console.log(data)` left in JS/TS code |
| VS-QLT-003 | Bare Except | `except:` catching all exceptions in Python |

### Example 2: Scan before every git push (CI/CD)

Add to your `.github/workflows/security.yml`:

```yaml
name: Security Scan

on: [push, pull_request]

jobs:
  vibesafe:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - run: |
          pip install click rich fastapi uvicorn jinja2 pyyaml
          git clone https://github.com/DohyeonLee-Dev/vibesafe-mvp.git /tmp/vibesafe
          python -m vibesafe.cli scan . --severity high --format json --output results.json
        working-directory: /tmp/vibesafe

      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: security-report
          path: /tmp/vibesafe/results.json
```

VibeSafe exits with code 1 if any critical or high issues are found, so the CI job will fail automatically.

### How the code works

1. **`cli.py`** — Defines three Click commands (`scan`, `dashboard`, `rules`). The `scan` command creates a `RuleEngine`, passes it to a `Scanner`, and formats the output.

2. **`scanner.py`** — Walks the directory tree recursively. For each file, it reads the content and passes it to the `RuleEngine`. It skips binary files, large files (>1MB), and common non-essential directories like `node_modules`.

3. **`rules.py`** — Contains 20+ `Rule` objects, each with a compiled regex pattern, severity level, file type filter, and fix hint. The `RuleEngine.check()` method runs every rule against every line of a file.

4. **`report.py`** — Takes a list of `Finding` objects and generates a complete HTML page with inline CSS. The report uses a dark theme, shows a letter grade, severity stats, and each finding with its code snippet.

5. **`web.py`** — Creates a FastAPI app that runs the scanner once at startup and serves the HTML report at `/` and JSON endpoints at `/api/findings` and `/api/summary`.
